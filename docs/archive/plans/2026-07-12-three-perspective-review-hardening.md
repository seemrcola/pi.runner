# 三视角 Review 收口实施计划

**目标：** 在不改变 Pi RUNNER 四层所有权模型的前提下，收口本轮架构、产品与视觉评审中有明确证据的问题。

**取舍：** 不重写稳定模块，不为未来需求预建抽象。只处理依赖方向、真实配置写入、断连交互、设置草稿保护、未使用脚手架和组件语言一致性。

## 修复项

1. 删除 `shared/chat.ts -> shared/protocol.ts` 的无调用方转导出，让 transport 依赖 domain 保持单向。
2. 将 Button variants 移出组件 barrel，消除 `Button.vue <-> index.ts` 运行时依赖环。
3. 在会话 intent 层阻止断连时创建本地占位会话，并统一空状态、弹窗和命令面板的禁用规则。
4. 设置页保留未保存草稿；脏状态下禁止刷新覆盖，并提供每个配置文件的显式还原动作。
5. Pi 配置通过同目录临时文件和原子 rename 写入；只有 `ENOENT` 被解释为文件不存在，其他读取错误上抛。
6. 删除没有调用方的 UI primitive 文件和无用类型依赖，保留实际使用的最小组件集合。
7. 移除全局自定义滚动条，收紧输入区圆角，使界面更接近原生桌面工具的克制组件语言。
